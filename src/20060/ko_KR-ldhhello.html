콤보

문제

당신은 액션 게임을 하고 있다. 컨트롤러에는 A, B, X, Y 버튼이 있고, 콤보 동작을 사용해 코인을 얻을 수 있다. 버튼을 순서대로 적절히 눌러 콤보 동작을 사용할 수 있다.

이 게임에는 숨겨진 콤보 동작 $S$가 있는데, 개발사에서 문자열 $S$는 공개하지 않지만 문자열의 길이 $N$은 공개하였다.

또한 콤보 문자열 $S$의 첫 번째 글자는 여러 번 등장하지 않는다고 한다. 예를 들어 콤보 문자열이 "ABXYY"이거나 "XYYAA"일 수 있지만, "AAAAA"나 "BXYBX"가 될 수는 없다고 한다.

당신은 매번 최대 $4N$개의 버튼을 눌러 볼 수 있다. 또한 당신이 누른 버튼의 문자열을 $p$라고 할 때, 당신은 $p$의 부분문자열 중 $S$의 가장 긴 접두어인 것의 길이만큼 코인을 얻을 수 있다.

예를 들어 문자열 $S$가 "ABXYY"이고 $p$가 "XXYYABYABXAY"인 경우, $p$의 부분문자열 중 $S$의 가장 긴 접두어는 "ABX"이기 때문에 당신은 3코인을 얻을 수 있다.

최소한의 콤보 연산만을 사용해 게임 제작사에서 알려주지 않는 콤보 문자열 $S$를 알아내 보자!

구현

당신은 아래 함수를 구현해야 한다.

string guess_sequence(int N)
  • N: 문자열 $S$의 길이.
  • 이 함수는 각 테스트케이스 당 정확히 한 번씩만 호출된다.
  • 이 함수는 콤보 문자열 $S$를 반환해야 한다.

위 함수는 다음 함수를 호출할 수 있다.

int press(string p)
  • p: 당신이 누른 버튼의 문자열.
  • p의 길이는 $0$ 이상 $4N$ 이하여야 하며, 모든 문자는 A, B, X, Y로 구성되어야 한다.
  • 각각의 테스트케이스마다 이 함수를 $8\,000$번을 초과해서 호출해서는 안 된다.
  • 이 함수는 p의 콤보를 넣었을 때 받을 수 있는 코인의 개수를 반환한다.

예시

문자열 $S$가 "ABXYY"라고 하자.
그레이더는 guess_sequence(5)를 호출하고, 아래 예시와 같은 함수 호출이 일어날 수 있다.

호출 반환
press("XXYYABYABXAY") 3
press("ABXYY") 5
press("ABXYYABXYY") 5
press("") 0
press("X") 0
press("BXYY") 0
press("YYXBA") 1
press("AY") 1

press 함수의 첫 번째 호출에서, "XXYYABYABXAY"의 부분문자열 중 "ABX"가 존재하지만, "ABXY"는 존재하지 않으므로 3이 반환된다.

press 함수의 세 번째 호출에서, "ABXYYABXYY"의 부분문자열로 "ABXYY가 등장하므로 5가 반환된다.

press의 여섯 번째 호출에서, "BXYY"의 부분문자열 중 "ABXYY"의 접두어가 존재하지 않으므로 0이 반환된다.

최종적으로 guess_sequence(5) 함수는 "ABXYY"를 반환해야 한다.

제한

  • $1 \le N \le 2\,000$
  • 문자열 $S$의 모든 문자는 A, B, X, Y 중 하나여야 한다.
  • 문자열 $S$의 첫 글자는 여러 번 등장해서는 안 된다.

서브태스크 1 (5점)

$N = 3$

서브태스크 2 (95점)

No additional constraints. For this subtask, your score for each test case is calculated as follows. Let be the number of calls to press.

추가 제약 조건이 없다. 이 서브태스크에서 당신의 점수는 아래 수식으로 계산된다. press 함수를 실행한 횟수를 $q$라고 하자.

  • $q \le N+2$인 경우, $95$점을 받는다.
  • $N+2 \lt q \le N+10$인 경우, $95 - 3(q - N - 2)$점을 받는다.
  • $N+10 \lt q \le 2N+1$인 경우, $25$점을 받는다.
  • $max{N+10, 2N+1} \lt q \le 4N$인 경우, $5$점을 받는다.
  • 이외의 경우, $0$점을 받는다.
  • if q ≤ N + 2, your score is 95.
  • if N + 2 < q ≤ N + 10, your score is 95 - 3(q - N - 2).
  • if N + 10 < q ≤ 2N + 1, your score is 25.
  • if max{N + 10, 2N + 1} < q ≤ 4N, your score is 5.
  • Otherwise, your score is 0.

노트

Note that your score for each subtask is the minimum of the scores for the test cases in the subtask.

샘플 그레이더

The sample grader reads the input in the following format:

  • line 1: S

If your program is judged as Accepted, the sample grader prints Accepted: q with q being the number of calls to the function press.

If your program is judged as Wrong Answer, it prints Wrong Answer: MSG. The meaning of MSG is as follows:

  • invalid press: A value of p given to press is invalid. Namely, the length of p is not between 0 and 4N, inclusive, or some character of p is not A, B, X, or Y.
  • too many moves: The function press is called more than 8,000 times.
  • wrong guess: The return value of guess_sequence is not S.
